==== Classify, Tag and Protect: A Holistic Approach to Data Governance in Snowflake Horizon ====
=== Speakers: ===
* Ankit Gupta
=== Notes: ===
* Intro
	* Importance of protecting sensitive data
		* Risk management
		* Compliance & Security
		* Privacy Protection
	* Governance of sensitive data: know (where?), monitor, protect, audit
	* You can use Snowsight UI or SQL Interfaces for classifying sensitive data.
		* SQL recommended for trying out this feature.
	* **Auto-classification is available in private preview currently.
	* Automatic Tag Propogation - tags can be configured to automatically apply to downstream objects
		* This is another private preview feature currently.
	* Access History
	* Data Governance UI
* Demo
	* See video for demo slides.
	* Demo setup is a medical company with a prod.core schema (patient/doctor/visit info), and a revenue_info schema. 
		* prod.core schema is copied into revenue_info, minus sensitive data.
		* Data steward would be managing prod.core, analyst would be working on revenue_info.
	* Demoed creating a custom classifier.
		* Custom tag created for specific fields. This tag is set to propogate in the demo. 
		* Create masking policy after this
		* Describe profile to review changes
		* SET classification profile on schema
			* Auto classification picks up that 2 tables need to have it applied
